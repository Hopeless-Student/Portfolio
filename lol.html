03 Process
On getting started
I usually overthink the beginning. But once I start, momentum builds — so I've learned to just open the file and do something, even if it's wrong. Course-correcting is easier than starting from scratch.
On losing motivation
It happens. I've stopped treating it like a character flaw. The better question is: what's the smallest possible thing I can do today to keep it moving? Usually that's enough.
On quality vs. done
I'd rather submit something honest and slightly rough than something polished that I don't fully understand. That said — I don't use "progress over perfection" as an excuse to be lazy. There's a difference.
On presenting work
I try to show the thinking, not just the output. A chart without context is just a picture. I'm still learning how to explain my reasoning clearly — but I know that's where the actual value is.
Where I'm at now
Building more, documenting better, getting comfortable with not knowing everything. Still early. That's fine.
